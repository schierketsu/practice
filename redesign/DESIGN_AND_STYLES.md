# Руководство по стилям и дизайну проекта «необманывай»

Документ описывает визуальную систему фронтенда (`frontend/`), чтобы перенести её в другие проекты: цвета, типографику, компонентные классы, сетку, адаптив и зависимости.

---

## 1. Стек и файлы

| Слой | Технология |
|------|------------|
| Фреймворк | Vue 3 (SFC) |
| Сборка | Vite 5 |
| Утилиты CSS | Tailwind CSS 3 (`@tailwind base/components/utilities`) |
| Кастомные стили | `frontend/src/style.css` — шрифты, `@layer base` / `@layer components`, медиазапросы |
| Конфиг темы | `frontend/tailwind.config.js` — расширение `theme.extend` |

Точка входа стилей: `frontend/src/main.js` импортирует `./style.css`.

---

## 2. Принцип дизайна (кратко)

- **Контраст шапки и контента**: верх страницы — тёмная полоса (`#212121`) с кремовым текстом (`#FFF5E5`); основной фон — белый (`#ffffff`).
- **Брендовая типографика**: заголовки и кнопки — шрифт **Polonium** (uppercase, широкий трекинг); основной текст — **JetBrains Mono** (в Tailwind задан как `font-sans` — осознанная «моноширинная» эстетика интерфейса).
- **Карточный UI**: белые блоки с рамкой `gray-200`, скругление `rounded-lg`.
- **Семантика оценок ИИ**: три уровня — зелёный / жёлтый / насыщенный розовый (не пастель), пороги `< 35%`, `35–70%`, `> 70%`.
- **Отчёт**: белая «оболочка» отчёта с **визуальным наездом** на нижнюю тёмную полосу (`report-footer-band`), футер сайта того же тона (`#2d2d2d`).

---

## 3. Палитра (основные цвета)

Используются как в Tailwind (`gray-*`, кастомные токены), так и «жёсткие» hex в CSS и классах.

### 3.1. Бренд и структура страницы

| Назначение | Hex | Где встречается |
|------------|-----|-----------------|
| Фон страницы / `#app` | `#ffffff` | `style.css`, `!important` на `html, body, #app, .app-shell` |
| Шапка (hero + загрузка) | `#212121` | `.header-dark`, кнопки в контексте шапки |
| Текст на шапке | `#FFF5E5` | `App.vue`, кнопки `.primary-btn` / `.secondary-btn` |
| Нижняя полоса под отчётом + футер | `#2d2d2d` | `.report-footer-band`, `.site-footer` |
| Hover кнопок на тёмном | `#2d2d2d` | переопределения в `main .content-block` и `.header-dark` |
| Разделитель шапки | `#333` | `border-[#333]` у `<header>` в `App.vue` |

### 3.2. Tailwind `theme.extend.colors` (`tailwind.config.js`)

| Токен | Значение | Назначение |
|-------|----------|------------|
| `ink` | `#111827` | Акцентный тёмный текст (класс `text-ink` в разметке) |
| `paper` | `#f9fafb` | Светлый фон (по задумке темы; страница принудительно белая) |
| `soft.*` | шкала gray-подобная | Расширенная палитра «мягких» оттенков |
| `accent.*` | серые | Вторичные акценты |
| `success` | `#059669` | Заложен в тему (в текущем UI доминируют traffic-light цвета для ИИ) |
| `warning` | `#d97706` | То же |
| `danger` | `#dc2626` | Ошибки; баннер ошибки — стандартные `red-50` / `red-200` |

### 3.3. Семантика «вероятность ИИ» (единая логика)

Пороги совпадают в JS (`aiChipClass`, заливка волны) и CSS-классах чипов:

| Уровень | Условие | Фон | Текст |
|---------|---------|-----|-------|
| Низкая | `< 35` | `#00FF00` | `#000` |
| Средняя | `35–70` | `#FFFF00` | `#000` |
| Высокая | `> 70` | `#FF1493` | `#fff` |

Дополнительно в деталях отчёта: вертикальные/горизонтальные чипы с заливкой через CSS-переменные `--ai-pct`, `--ai-fill`; нейтральный фон полосы `#e5e7eb`; для «нет данных» волна — `#D1D5DB`, подпись серая (`wave-strip-label--na`).

### 3.4. Календарь и «жёсткие» рамки метаданных

- Рамка календаря и секций в панели изображения: `border-[#212121]`, фон ячеек выделения: `bg-[#212121] text-[#FFF5E5]`.
- Зона загрузки: пунктир `border-[#212121]`, фон `#FFFFFF` / при drag `bg-gray-100`.

## 4. Типографика

### 4.1. Шрифты

1. **JetBrains Mono** — подключается в `index.html` с Google Fonts (`weights 400–700`). В `tailwind.config.js` задан как **`fontFamily.sans`** → класс `font-sans` даёт моноширинный UI.
2. **Polonium** — локальные файлы:
   - `frontend/src/public/Polonium/Polonium.otf` (400)
   - `frontend/src/public/Polonium/Polonium-Bold.otf` (700)  
   Подключение: `@font-face` в `style.css` (пути относительно `src/style.css`).

### 4.2. Глобальные правила заголовков (`@layer base` в `style.css`)

Для `h1–h6` по умолчанию:

- `font-family: 'Polonium', serif`
- `font-weight: 700`
- `letter-spacing: 0.1em`
- `text-transform: uppercase`
- Размеры от `h1` (3rem) до `h6` (1.125rem)

**Исключение для контента отчёта**: в `main .content-block` и `main .report-view-shell` заголовки `h3–h6` сбрасываются на «подписи»: обычный шрифт (наследование), `font-weight: 400`, без uppercase, цвет `text-gray-500` по смыслу (в CSS `#6b7280`), размер `0.875rem`.

### 4.3. Класс `font-polonium`

Используется в разметке для крупных заголовков бренда и секций, иногда с `normal-case` / `tracking-normal` там, где не нужен дефолтный uppercase из глобальных `h*`.

### 4.4. Моноширинные значения

Метаданные и технические поля: `font-mono` (наследует JetBrains Mono через `sans` stack — фактически тот же семейный акцент).

## 5. Макет и отступы

### 5.1. Оболочка приложения

- **`.app-shell`**: `min-h-screen flex flex-col overflow-x-hidden`, фон белый.
- **Контент**: `mx-auto max-w-6xl px-3 sm:px-6` — основной горизонтальный ритм; вертикальные отступы варьируются по состоянию (есть ли результат анализа, идёт ли загрузка).

### 5.2. Tailwind breakpoints (используемые в проекте)

- `sm`: 640px — усиленные отступы, сетка авторов, крупнее типографика кнопок в медиазапросе `<640px` в `style.css`.
- `lg`: 1024px — двухколоночный блок результатов (`results-two-column`: сайдбар 280px + контент).

### 5.3. Полноширинная полоса

- **`.full-bleed`**: `100vw` с отрицательными margin через `calc(50% - 50vw)` — для фона под отчётом на всю ширину viewport.

В `@media (max-width: 639.98px)` у кнопок уменьшаются padding, шрифт, margin; добавлен `max-width: 100%`.

## 7. Поведение и микроанимации

- **Волна в шапке деталей** (`ReportView.vue`): SVG path пересчитывается в `requestAnimationFrame` (`buildWavePath`, период ~4 с). Цвет заливки синхронизирован с порогами ИИ (те же hex, что у чипов).
- **Спиннер загрузки**: `animate-spin` + кольцо `border-gray-200` / `border-t-gray-700`.
- **Тултип** (иконка помощи): `group-hover` / `group-focus-visible`, тёмный фон `bg-gray-900`, белый текст, тень.

В `index.html` есть скрытый SVG с `clipPath` для волны (`waveClipBox`) — на текущую отрисовку в `ReportView` влияет путь из скрипта; defs могут использоваться для будущих экспериментов или наследия.

## 8. Зависимости для переноса в новый проект

### 8.1. npm

```json
"devDependencies": {
  "tailwindcss": "^3.3.5",
  "postcss": "^8.4.31",
  "autoprefixer": "^10.4.16"
}
```

Нужны файлы: `tailwind.config.js`, `postcss.config.js` (стандартный postcss + tailwind + autoprefixer).

### 8.2. Обязательные артефакты

1. Скопировать **`frontend/src/style.css`** (или вынести общие токены в CSS variables позже).
2. Скопировать **`frontend/tailwind.config.js`** и подправить `content` под пути нового проекта.
3. Подключить шрифты: ссылка JetBrains Mono в HTML + файлы **Polonium** с корректными путями в `@font-face`.
4. Статика: `logo.png`, `iconhelp.png` из `frontend/public/` (или эквиваленты).

### 8.3. Чеклист переноса разметки

- Обернуть приложение в `.app-shell`.
- Шапку с загрузкой дать классы `header-dark` + нужные `border-b`.
- Основной контейнер: `max-w-6xl` + согласованные `px/py`.
- Карточки: `card`, заголовки секций — `font-polonium` + размеры как в `App.vue` / `ReportView.vue`.
- Кнопки действий в брендовом стиле: классы `primary-btn` / `secondary-btn` (на тёмном фоне читаются за счёт кремовой обводки и текста).
- Для отчёта воспроизвести связку: `report-view` → `report-view-shell` → в конце `report-footer-band full-bleed`, футер с `.site-footer`.

## 9. Согласованность порогов и подписей

В одном месте зафиксируйте константы и используйте везде:

- **Чипы / CSS**: `status-chip-ai-low` &lt; 35, `status-chip-ai-mid` 35–70, `status-chip-ai-high` &gt; 70.
- **Текст волны**: «низкая» / «средняя» / «высокая» при тех же границах; «нет изображений» — отдельное состояние с серой заливкой и классом `wave-strip-label--na`.

## 11. Расширение и кастомизация

- Вынести повторяющиеся hex (`#212121`, `#FFF5E5`, `#2d2d2d`, traffic-light) в **CSS variables** на `:root` и подставить в Tailwind через `theme.extend.colors` — упростит смену бренда.
- Если в новом проекте не нужна моноширинная основа текста, замените `fontFamily.sans` на гуманистический sans (например Inter) и оставьте Polonium только для `.font-polonium` и кнопок.
- Классы `success` / `warning` / `danger` в `tailwind.config.js` сейчас почти не задействованы в UI оценок ИИ — их можно использовать для форм, toast и статусов backend.
